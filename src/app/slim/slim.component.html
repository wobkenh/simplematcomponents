Plain table with Virtual Scroll

If you have a lot of data you need to display at once because people are screaming at you that they do not like pagination, you can use smc-simpletable as an alternative to smc-simplemattable. It uses virtual scrolling from the cdk package to only render data visible in the current viewport. smc-simpletable can easily handle 10000 rows or more at once. The interface of both simplemattables is mostly identical, so it often can be used as a drop in replacement, but some features are not supported and some things need to be noticed when using the slim version:

  • simpletable always needs to get a height assigned from the outside. This can be a fixed height or e.g. a container with flex that uses e.g. the window height to fill available space
  • Pagination is not supported (for obvious reasons)
  • Sorting is always enabled
  • Each row, including the header row, must always have the same height. Variable height rows are not supported. You can control the height using the itemSize parameter and control the item buffer using the pageSize parameter. One exception is the detail row component for expandable rows, which is supported and may take a variable amount of space
  • Buttons in table cells are supported, but you need to accommodate for their size in the itemSize parameter.

Resizable Plain Table with Virtual Scroll

If even with the virtual scroll people are still screaming at you because they cannot resize the table colums the same way they could in their windows 98 application they used for the last 25 years, you can set the [resizeableColumn] parameter to true. This will set the table layout to fixed and make every column resizable using a handle bar on the right side of each cell. In contrast to table layout auto, table layout fixed will not calculate the column widths according to the content. So if you want anything but an equal distribution of the column widths, you need to set them yourself.
